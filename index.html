git clone

Descárgate el repositorio en una carpeta local. Analiza en qué rama te encuentras y el total de ficheros que visualizas en dicha rama

git branch

Este repositorio tiene varias ramas cada una con distintos commits. Lístalas e indica qué significa el asterisco

git checkout

Cambia de rama y fíjate en el sistema de archivos (working tree) cómo aparecen y desaparecen los ficheros. Siguen estando ahí, pero en cada rama los contenidos son distintos: piensa en las ramas como mundos paralelos

git add/git commit

En cada rama debes hacer una modificación: añadir pantallazos e introducirlos en el este fichero index. Añade la información que precises para completar cada apartado de este fichero.

git rm

En una de las ramas hay muchos ficheros temporales que debes borrar. Asegúrate de que sólo borras los imprescidibles

git

Tras realizar los distintos commits debes fusionar cada rama a la rama principal.

git log --oneline --decorate --graph --all

Muestra el estado de los commits de forma gráfica

git checkout -b

Crea una rama nueva y añade contenido relacionado con git
